# 菜单搜索收藏视图设计

## 背景与问题

全局菜单搜索在无关键词时只展示最近访问。界面虽然允许通过标签菜单或搜索结果右侧星标进行收藏，但没有收藏集合入口，用户无法建立“收藏后去哪里找”的稳定心智。

当前实现还存在更深一层的数据边界问题：收藏值挂在临时 `Tab.favorite` 上。未打开成标签的菜单无法真正收藏，关闭标签或清理标签状态也会同时丢失收藏。若只增加一个列表，会把这个缺陷直接暴露给用户。

## 目标

1. 在既有 `Ctrl/Cmd + K` 菜单搜索入口内提供可发现的收藏列表。
2. 同时支持鼠标点击和键盘快速切换，并记住用户最后选择的默认列表。
3. 让菜单收藏脱离标签打开、关闭与恢复生命周期。
4. 收藏与最近访问只展示当前账号可访问的菜单，并使用当前语言的标题和图标。
5. 保持搜索态单一、清晰，不把快速导航弹层扩展成菜单管理器。

## 方案比较

### 方案 A：收藏与最近访问混排

实现成本最低，但收藏和历史会重复，排序规则难解释，用户也无法只查看收藏。该方案不满足“可切换并记忆”的核心诉求。

### 方案 B：独立收藏页或抽屉

适合排序、分组、批量管理等重型能力，但当前需求只是快速找回收藏。新增入口会割裂全局搜索路径，并引入额外导航层级。

### 方案 C：搜索弹层内的双集合视图

在无关键词状态下展示“最近访问 / 收藏菜单”分段控制器；点击或按 `← / →` 切换，保存最后选择。输入关键词后隐藏集合控制器，只呈现搜索结果。

采用方案 C。它在现有入口内补齐收藏闭环，交互成本低，也为后续扩展数量或排序保留清晰边界。

## UI / UE / UX 设计

### 信息结构

- 搜索输入区保持第一视觉层级。
- 无关键词时，在输入区下方增加紧凑工具栏：左侧为两个等权集合选项，右侧只在“最近访问”有内容时显示“清空”。
- 集合选项展示图标、名称与数量，让收藏入口及其状态可被立即发现。
- 结果行继续复用当前图标、标题、路径、星标与 Enter 提示，不引入新的卡片层级。
- 底部快捷键提示在无关键词时增加 `← / → / 切换分类`。

### 状态与交互

| 条件 | 主内容 | `← / →` 行为 | `Tab` 行为 | Enter 行为 |
| --- | --- | --- | --- | --- |
| 无关键词 + 最近访问 | 当前可访问的最近菜单 | 在最近/收藏之间循环切换 | 原生焦点移动 | 打开高亮菜单 |
| 无关键词 + 收藏菜单 | 当前可访问的收藏菜单 | 在收藏/最近之间循环切换 | 原生焦点移动 | 打开高亮菜单 |
| 有关键词 | 搜索结果 | 原生文本光标移动 | 原生焦点移动 | 打开高亮结果 |

- 点击分段选项或在空搜索框按 `← / →` 后，活动行回到第一项，列表滚动回顶部。
- `Tab` 与 `Shift + Tab` 始终保持原生焦点行为，不再承担分类切换。
- 收藏列表中取消收藏后，该行即时消失；活动下标自动收敛到剩余范围。
- 搜索框清空后回到用户已记忆的集合，不强制回退最近访问。
- 外部链接使用新标签页打开，内部菜单继续通过 Vue Router 导航。

### 空状态

- 最近访问为空：标题“暂无最近访问”，说明“访问过的菜单会显示在这里”。
- 收藏为空：标题“暂无收藏菜单”，说明“点击菜单右侧的星标即可收藏”。
- 搜索无结果：沿用现有“未找到匹配的菜单”。

空状态使用现有图标、主题变量与排版密度，不加入插画或大面积装饰。

### 可访问性

- 弹层声明为模态对话框，并提供本地化名称。
- 分段控制器使用同组 radio 语义，可通过方向键操作。
- 菜单主操作使用真实按钮；方向键改变活动项时通过礼貌级 live region 宣告当前菜单。
- 星标按钮提供随状态变化的 `aria-label` 与 tooltip 文案。
- 搜索输入保留自动聚焦，Esc 徽标本身也是可点击的关闭按钮；分类快捷键只在输入为空时生效，不影响文本光标与 Tab 导航。
- 清空历史或从收藏列表移除聚焦项后，焦点回到搜索输入；焦点陷阱也会接管意外落到对话框外的焦点。
- 移动端隐藏纯键盘提示；焦点指示器使用高对比文本 token，兼容亮色、暗色和自定义主题色。

## 前端架构设计

### 数据边界

新增独立的菜单偏好 store，持有：

- `favoritePaths: string[]`：按收藏顺序保存菜单身份。
- `searchView: 'recent' | 'favorites'`：保存菜单搜索默认集合。

只持久化 path，不持久化标题、图标或权限信息。展示数据始终从 `permissionStore.menuTree` 派生的当前 `searchSource` 中按 path 选择，因此语言切换、路由元数据更新和权限变化可以立即生效。

### 持久化与迁移

本期沿用主题、布局等偏好的浏览器本地语义，不引入账号或服务端同步；收藏与最近访问的可见内容仍严格取当前权限菜单的交集，避免在切换账号时暴露不可访问入口。若产品未来明确支持共享设备上的多账号偏好隔离，再将存储 key 按用户标识分区。

- 收藏使用 `app-menu-favorites`。
- 默认集合使用 `app-menu-search-view`。
- 首次读取新收藏 key 时，若不存在有效数组，则从旧 `app-tabs-state.tabs[].favorite` 提取 path 并写入新 key。
- 损坏 JSON、非法视图值和重复 path 均安全回退，不影响应用启动。
- `Tab.favorite` 暂时保留为旧状态迁移字段，但新代码不再写入。

### 组件职责

- `menuPreferences` 纯函数模块：解析、规范化、旧数据迁移和按 path 选择当前菜单，便于 Node 环境单测。
- `menuPreferences` Pinia store：提供响应式收藏与视图动作，并封装 localStorage 读写失败。
- `GlobalSearch.vue`：负责集合工具栏、键盘状态机、当前权限菜单交集、渲染和导航。
- `TabBar.vue`：保留原收藏入口，但改为调用菜单偏好 store。
- `tabs.ts`：回归标签打开、关闭、固定和恢复职责，不再管理收藏动作。

### 权限与陈旧数据

收藏列表和最近访问列表都通过当前 `searchSource` 取交集。无权限、已删除或隐藏的 path 不展示，也不会被直接导航；收藏 path 本身暂不删除，以便同一浏览器后续恢复权限时仍能找回偏好。

## 验证策略

- 纯函数单测覆盖收藏解析、旧数据迁移、非法数据回退、视图白名单和权限菜单交集排序。
- 运行完整单测、lint、类型检查、格式检查和生产构建。
- 浏览器验收覆盖点击切换、左右方向键切换、原生 Tab 导航、选择记忆、未打开菜单收藏、关闭标签后收藏保留、取消收藏即时移除、清空历史、搜索态、外链、亮/暗主题及控制台错误。
