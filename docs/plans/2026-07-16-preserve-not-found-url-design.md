# 保留原始 URL 的 404 路由设计

## 目标

访问不存在的路由时保持浏览器 URL 不变，并直接渲染现有 404 页面。

## 设计

catch-all 路由直接挂载 `src/views/error/404.vue`，不再重定向到 `/404`。显式 `/404` 路由继续保留，兼容已有入口。

动态权限路由尚未生成时，catch-all 可能暂时匹配一个实际存在的受权限路由。导航守卫通过 `NotFoundCatchAll` 路由名识别这种情况：已登录且权限路由未生成时，先生成动态路由，再使用原始 `fullPath` 重试。若重试后仍匹配 catch-all，则原地址直接显示 404。

未登录访问未知地址时不再跳转登录页，因为未知地址不应被假定为受保护路由；它在原地址显示 404。

## 验证

- 契约测试确认 catch-all 使用 404 组件且没有 redirect。
- 契约测试确认动态路由恢复判断基于 catch-all 路由名和生成状态。
- 运行目标单测、类型检查、lint 和生产构建。
