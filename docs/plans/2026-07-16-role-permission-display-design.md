# 角色权限展示与回显修复设计

## 目标

修复角色管理编辑弹窗中权限名称显示为 `[object Object]`、角色默认权限为空导致权限数量和编辑回显不正确的问题，使权限树、多语言展示、列表数量、编辑回显和提交数据保持一致。

## 根因

- 权限名称允许为 `string | LocalizedText`，角色页却通过模板字符串直接拼接 `node.name`，多语言对象因此被转换为 `[object Object]`。
- `mockRoles` 中所有角色的 `permissions` 都是空数组，列表权限数和编辑表单只能如实显示为 0；这不是 TreeSelect 的计数错误。
- 权限页已有局部的多语言名称解析逻辑，但未形成可复用契约，角色页无法共享。

## 架构

- 将多语言文本解析提取为纯工具函数，按当前语言、简体中文、英文、首个可用值的顺序回退。
- 权限页和角色页统一使用该工具函数，避免同类页面继续产生不同展示结果。
- mock 角色从 `mockPermissions` 权限树按权限代码规则派生权限集合，不手写重复的权限对象或易漂移的权限 ID。
- Administrator 获得全部权限；Manager 获得仪表盘和组织管理权限；User 获得仪表盘及基础只读权限；Guest 仅获得仪表盘权限。
- 角色保存仍提交完整的 `Permission[]`，不改变现有 API 和 ProForm/TreeSelect 契约。

## 行为边界

- 不修改公共 TreeSelect 去识别业务多语言对象；`ProFormOption.label` 仍必须是可展示文本。
- 角色权限数量按实际绑定的权限节点数量计算，父节点和子节点均作为独立权限节点计数，与当前 `permissions.length` 语义一致。
- mock 角色复用权限数据的克隆结果，避免角色编辑反向修改全局权限树。
- 对未知语言保持稳定回退，不显示对象字符串或空白标签。

## 验证

- 单元测试覆盖多语言文本当前语言选择和回退规则。
- 单元测试覆盖各 mock 角色权限集合、数量和对象独立性。
- 运行定向单测、type-check、lint 和 build:check。
- 浏览器验证中文与英文下权限名称、角色列表数量及编辑回显。
