# 项目主题色一致性审计设计

## 审计结论

项目的 Antdv Next 主题通过 `App.vue` 中的 `ConfigProvider` 响应 `settingsStore.primaryColorHex`，自定义组件则主要消费 `--color-primary` 及其色阶。整体链路有效，但仍有两处同类问题：

1. 自定义主题会把主色和 1–10 色阶写入根节点内联样式。切回预设主题时，这些内联变量没有清除，会覆盖 `data-primary-color` 对应的样式表变量。
2. 滑块验证码的待验证交互轨道使用固定信息色，而同组点选、旋转验证码均使用主题主色。

成功、警告、错误、信息状态以及分类标签、文件类型、图表系列、代码示例的固定颜色具有独立语义，不纳入主题色替换范围。

## 设计

- `setPrimaryColor` 在启用预设主题前清除自定义主色和 1–10 色阶的根节点内联声明，让 CSS 预设重新成为变量来源。
- 保留 `--ant-primary-color` 兼容变量，并继续写入当前预设色。
- 滑块验证码的待验证轨道改用 `--color-primary-bg` / `--color-primary`，成功状态继续使用成功色。
- 增加 store 回归测试，验证从自定义色切换到预设色时恰好清除所有自定义主题变量。

## 验证

- 定向运行 settings store 单测。
- 运行主题相关既有单测、类型检查、lint、修改文件格式检查和生产构建。
- 检查 diff，避免纳入角色与权限模块的既有未提交修改。
