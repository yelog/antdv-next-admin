# 登录状态感知的 404 页面设计

## 目标

统一显式 `/404`、未知地址和示例菜单中的 404 内容，并根据用户当前登录状态决定页面承载方式：已登录时保留后台左侧菜单、顶部栏和标签栏，未登录时全屏展示。

## 方案

新增公共 `NotFoundContent.vue`，只负责渲染 404 状态、说明文字和返回首页按钮。全局错误页与示例页均复用该组件，避免两套内容和样式继续分化。

全局 `views/error/404.vue` 读取认证 store 的 token：

- 没有 token 时，使用全屏居中容器渲染公共内容。
- 存在 token 时，使用 `AdminLayout` 承载公共内容，以显示后台导航框架。

为避免在错误页中创建第二层路由，`AdminLayout` 提供一个默认内容插槽；有插槽内容时直接渲染插槽，没有时保持现有 `<router-view>` 行为。示例菜单的 404 已位于 `AdminLayout` 的子路由中，因此只需在现有页面容器内复用公共内容，不重复嵌套布局。

## 路由与数据流

`/404` 与 `/:pathMatch(.*)*` 继续指向全局错误页，并保留未知地址的原始 URL。路由守卫仍在有 token 且权限路由尚未生成时优先恢复动态路由，再重新解析原地址；只有确认没有匹配路由后才渲染 404。

登录状态以 `authStore.token` 为唯一判断依据，与现有路由守卫保持一致。返回首页按钮统一跳转 `/`。

## 测试

- 验证用于选择 404 显示模式的逻辑：有 token 使用后台布局，无 token 使用全屏模式。
- 保留并运行动态路由恢复测试，确保合法动态地址刷新时不会误显示 404。
- 运行类型检查、单元测试、lint 和生产构建，确认布局插槽没有影响普通业务路由。
